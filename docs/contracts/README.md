# Contract and Proof Samples

This directory contains A/B-track fixtures for the VeriVote aggregator, ZK, and chain demo.

## Files

- `valid_vote_records_8x4.sample.json`: fixed 8-ballot, 4-candidate witness fixture.
- `demo_seed_fixture.json`: minimal 4-candidate/8-vote seed plan for independent API/UI demos.
- `VERIVOTE_PARALLEL_INTERFACE_CONTRACT.md`: frozen A/B/C/D field and sample contract.
- `aggregator_report_v2.sample.json`: AggregatorReport v2 fixture with partitionAudit, explicit `validVoteIds` / `invalidVoteIds`, per-bucket `tokenHashes`, diagnosticsHash, publicInputHints, Pedersen aggregate metadata, `proofStatus=not-generated`, `tallyProofSummary`, and `integrityCheck`.
- `aggregator_report.sample.json`: day-0 compatibility alias for teams that still consume the original sample filename.
- `aggregator_report_pedersen_null.sample.json`: C-track-not-ready fixture where `pedersenAggregateStatus=pending`, `pedersenAggregateAudit`, `pedersenAggregateHash`, and `publicInputHints.pedersenAggregateHash` are `null`, but `auditHash` and `integrityCheck` still verify.
- `export_bundle_v2.sample.json`: full ExportBundleV2 sample with envelope, election, publicInputs, bulletinBoard, aggregatorReport, zkSummary, tallyProofSummary, chainAudit, challengeRecords, and demoMetadata.
- `public_inputs_v2.sample.json`: public-input artifact exported from the v2 report.
- `../evaluation/aggregator_reports/summary.json`: A-track normal/attack acceptance summary generated by `pnpm aggregator:audit-cases`.
- `../evaluation/aggregator_reports/offline_verification.json`: A-track disk-based verification and tamper rejection summary generated by `pnpm aggregator:verify`.
- `../evaluation/aggregator_reports/api_smoke.json`: A-track HTTP smoke summary generated by `pnpm aggregator:api-smoke`.
- `../evaluation/aggregator_reports/api_export_aggregator_report.json`: raw HTTP export from `/elections/:id/export/aggregator_report.json`.
- `../evaluation/aggregator_reports/api_export_public_inputs.json`: raw HTTP export from `/elections/:id/export/public_inputs.json`.
- `../evaluation/aggregator_reports/api_export_bundle.json`: raw HTTP export from `/elections/:id/export-bundle`.
- `../evaluation/aggregator_reports/api_aggregator_report.attack-tamper-tally.json`: saved AggregatorReport tamper negative case where `auditHash` is recomputed but integrity and tally consistency still fail.
- `../evaluation/aggregator_reports/local_standalone/manifest.json`: no-server local export manifest generated by `pnpm aggregator:local-export`.
- `../evaluation/aggregator_reports/local_standalone/aggregator_report.local-normal.json`: standalone AggregatorReport v2 generated without starting the API server.
- `../evaluation/aggregator_reports/local_standalone/public_inputs.local-normal.json`: standalone public inputs generated without starting the API server.
- `../evaluation/aggregator_reports/local_standalone/export_bundle.local-normal.json`: standalone ExportBundleV2 generated without starting the API server.
- `../evaluation/aggregator_reports/powershell_api/manifest.json`: PowerShell Invoke-RestMethod smoke manifest generated by `pnpm aggregator:ps-smoke`.
- `../evaluation/aggregator_reports/powershell_api/aggregator_report.normal.json`: normal AggregatorReport v2 saved by the PowerShell API path.
- `../evaluation/aggregator_reports/powershell_api/aggregator_report.attack-*.json`: attack AggregatorReport v2 files saved by the PowerShell API path.
- `../evaluation/aggregator_reports/powershell_api/public_inputs.normal.json`: public inputs saved through PowerShell API calls.
- `../evaluation/aggregator_reports/powershell_api/export_bundle.normal.json`: ExportBundleV2 saved through PowerShell API calls.
- `../evaluation/aggregator_reports/python_api_smoke.json`: Python/FastAPI parity smoke for AggregatorReport v2 and ExportBundleV2.
- `../evaluation/aggregator_reports/python_api_aggregator_report.json`: raw Python API AggregatorReport v2 response.
- `../evaluation/aggregator_reports/python_api_public_inputs.json`: raw Python API public inputs v2 response.
- `../evaluation/aggregator_reports/python_api_export_bundle.json`: raw Python API ExportBundleV2 response.
- `../evaluation/aggregator_reports/api_schema_parity.json`: TypeScript/Python API schema parity check for AggregatorReport v2, public inputs, partition buckets, diagnostics, `integrityCheck.checks`, and ExportBundleV2.
- `../evaluation/aggregator_reports/task_a_traceability.json`: A-01 to A-20 source/evidence/command/API traceability matrix generated by `pnpm aggregator:complete`.
- `../evaluation/aggregator_reports/completeness_matrix.json`: A-track deliverable completeness gate generated by `pnpm aggregator:complete`, including frontend checks that the Aggregator/Audit pages expose proof placeholder, Pedersen state, public input hints, vote-id accounting, and bucket token hashes.
- `tally_proof_v2.valid.sample.json`: real Groth16 TallyProof v2 generated after `pnpm zk:setup`.
- `tally_proof_v2.invalid-tally.sample.json`: invalid request fixture where tally does not match the witness.
- `calldata.sample.json`: real Solidity calldata `{ a, b, c, input }` derived from the valid proof.
- `chain_audit.real.sample.json`: local Hardhat transaction sample using the generated `TallyVerifier`.

## Real/Mock Boundary

The valid proof and calldata files in this directory are real Groth16 artifacts
for the current local `zk-artifacts` and `contracts/TallyVerifier.sol`.

If `pnpm zk:setup` is run again, the zkey and verifier constants change; refresh
these samples before using them in tests or demos. The Hardhat real-verifier
test defaults to `docs/contracts/calldata.sample.json`; override it with
`VERIVOTE_REAL_TALLY_CALLDATA_JSON` only when testing another fixture.

## Reproduce

```bash
pnpm aggregator:audit-cases
pnpm aggregator:verify
pnpm aggregator:api-smoke
pnpm aggregator:local-export
pnpm aggregator:ps-smoke
pnpm aggregator:complete
pnpm zk:setup
pnpm zk:samples
pnpm zk:audit
pnpm contract:test
pnpm --filter @verivote/contracts run sample:chain-audit
```

`pnpm aggregator:audit-cases` is the A-track gate: it regenerates the v2
aggregator report and ExportBundleV2 samples, recomputes the report integrity
checks, verifies all normal/attack expectations, validates explicit vote-id
coverage, proof placeholder fields, and bucket token roots, and refreshes the
acceptance summary.
`pnpm aggregator:verify` is the A-track offline gate: it reloads the generated
JSON evidence from disk and checks both positive reports and negative tamper
cases, including malformed shape, vote-id tampering, bucket token tampering, and
duplicate-token diagnostic drift.
`pnpm aggregator:api-smoke` is the A-track HTTP and persistence gate: it starts
the API in memory mode, seeds fixture elections, exercises run/report/export/
attack endpoints including `/elections/:id/export-bundle`, restarts the API in
SQLite mode, and compares every AggregatorReport v2 persistence field before
and after restart. It also checks the `tamper-tally` saved-report negative case,
where `auditHashMatches=true` but tally/bucket/public-input consistency checks
fail. The field-level result is written to `api_smoke.json`.
`pnpm aggregator:local-export` is the A-track no-server gate: it creates the
same 4-candidate/8-vote fixture in memory, emits AggregatorReport v2,
publicInputs v2, ExportBundleV2, and a manifest under
`docs/evaluation/aggregator_reports/local_standalone/`, and verifies the report
integrity before writing files.
`pnpm aggregator:ps-smoke` is the A.5 no-frontend API path for PowerShell
reviewers. With the API already running, it uses `Invoke-RestMethod` to seed
normal and attack elections, run the aggregator, and save `aggregator_report.*`,
`public_inputs.normal.json`, `export_bundle.normal.json`, and `manifest.json`
under `docs/evaluation/aggregator_reports/powershell_api/`.
`python scripts/api_smoke_test.py` is the Python/FastAPI parity gate. It checks
that the secondary API entrypoint emits the same AggregatorReport v2,
publicInputs v2, ExportBundleV2 surface, and attack diagnostics including
duplicate-token, invalid-candidate, invalid-one-hot, and candidate-vector-mismatch.
`pnpm aggregator:complete` checks the A-track deliverable set itself, including
contract docs, handoff docs, generated samples, offline tamper evidence, HTTP
smoke evidence, no-server local export evidence, PowerShell API evidence,
`aggregator_report.attack-*.json` compatibility outputs, and field-level
persistence evidence. It also statically checks the frontend report views so the
A-track v2 evidence is visible in the UI, not only present in JSON.
